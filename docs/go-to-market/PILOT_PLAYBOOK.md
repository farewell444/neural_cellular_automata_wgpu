# CellForge Pilot Playbook

## Pilot Goal
Prove that the CellForge SDK provides production value quickly with low integration risk.

## Ideal Pilot Profile
- UE5 project already in active development
- needs dynamic damage/healing or reactive materials
- has one technical owner and one creative owner

## Pilot Scope (Recommended)
- duration: 2 to 4 weeks
- one map or one gameplay feature
- one to two CellForge presets
- one quality profile target

## Pilot Success Metrics
- integration time <= 2 days
- stable runtime in target scene
- no critical crash during pilot
- measurable visual or production gain

## Discovery Checklist
1. Which scene or feature needs dynamic regenerative behavior?
2. What is target platform and GPU class?
3. What frame-time budget is available?
4. Is this VFX-driven or gameplay-critical?
5. What does "pilot success" mean for this team?

## Delivery Workflow
1. Kickoff and requirement lock (Day 1)
2. Integration and baseline profiling (Days 2-5)
3. Preset tuning and gameplay hook-up (Days 6-10)
4. Stability pass and optimization (Days 11-14)
5. Final review and decision (end of pilot)

## Support Protocol
- response SLA:
  - critical issue: same business day
  - major issue: next business day
  - minor issue: within 3 business days
- weekly sync call
- issue tracker with severity labels

## Commercial Conversion Trigger
A pilot is considered conversion-ready when all are true:
- integration completed in agreed scope
- performance meets agreed profile
- team confirms production intent for the feature

## Suggested Pilot Pricing
- fixed-price pilot with clear scope
- discount applied toward annual SDK license if converted

## Deliverables at Pilot End
- integration branch or package
- performance report
- risk register and mitigation notes
- recommended production rollout plan
- commercial proposal for continuation
